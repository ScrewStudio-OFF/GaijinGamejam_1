# WASD Управление игроком в EdenSpark

## Как это работает

Система разбита на три части: **инициализация**, **получение ввода** и **обновление позиции**.

---

## 1. ИНИЦИАЛИЗАЦИЯ УПРАВЛЕНИЯ (`on_initialize`)

```daslang
add_and_activate_action_set("playerInput", {
    "move" => Action(
        joystickActions = JoystickAction(
            keyboardDPads = [
                KeyboardDPadInput(
                    leftIdx = KeyCode.A,    // A = влево (x-)
                    rightIdx = KeyCode.D,   // D = вправо (x+)
                    upIdx = KeyCode.W,      // W = вперёд (z+)
                    downIdx = KeyCode.S     // S = назад (z-)
                )
            ]
        )
    )
})
```

### Что здесь происходит:

- **`add_and_activate_action_set()`** — создаёт систему управления "playerInput"
- **`Action()`** — определяет действие "move"
- **`KeyboardDPadInput()`** — отображает четыре клавиши на D-Pad (крестовину управления)
  - `upIdx = KeyCode.W` — нажатие W отправляет сигнал "вверх"
  - `downIdx = KeyCode.S` — нажатие S отправляет сигнал "вниз"
  - `leftIdx = KeyCode.A` — нажатие A отправляет сигнал "влево"
  - `rightIdx = KeyCode.D` — нажатие D отправляет сигнал "вправо"

---

## 2. ПОЛУЧЕНИЕ ВВОДА (`on_update`)

```daslang
let move_input = get_action_vector2("move")
// move_input: float2
// x: -1 (A pressed) ... 0 (nothing) ... +1 (D pressed)
// y: -1 (S pressed) ... 0 (nothing) ... +1 (W pressed)
```

### Что возвращает `get_action_vector2()`:

| Нажато | move_input |
|--------|-----------|
| Ничего | (0, 0) |
| W | (0, +1) |
| S | (0, -1) |
| A | (-1, 0) |
| D | (+1, 0) |
| W+D | (0.707, 0.707) - диагональ |
| W+A | (-0.707, 0.707) - диагональ |

---

## 3. ПРЕОБРАЗОВАНИЕ 2D ВВОДА В 3D ДВИЖЕНИЕ

```daslang
// Преобразуем 2D ввод в 3D направление
var move_direction = float3(move_input.x, 0.0, move_input.y)
// float3:
//   x = горизонтальное движение влево/вправо
//   y = 0 (нет прыжка, только земля)
//   z = движение вперёд/назад
```

### Почему так:

- **X-ось**: `move_input.x` — элемент A/D управления
- **Y-ось**: `0` — игрок ходит по земле, не летает
- **Z-ось**: `move_input.y` — элемент W/S управления

---

## 4. НОРМАЛИЗАЦИЯ НАПРАВЛЕНИЯ

```daslang
if (length(move_direction) > 0.0) {
    move_direction = normalize(move_direction)
    velocity = move_direction * MOVE_SPEED
}
```

### Почему нормализация нужна:

**Без нормализации:**
- W alone: `(0, 0, 1)` → длина = 1
- W+D diagonal: `(1, 0, 1)` → длина = √2 ≈ 1.41
- **Диагональ быстрее чем прямое движение!**

**С нормализацией:**
- W alone: `normalize((0, 0, 1))` = `(0, 0, 1)` → длина = 1
- W+D diagonal: `normalize((1, 0, 1))` = `(0.707, 0, 0.707)` → длина = 1
- **Все направления одинаковая скорость**

---

## 5. ПРИМЕНЕНИЕ ТРЕНИЯ

```daslang
} else {
    // Если клавиши не нажаты
    velocity = velocity * FRICTION  // FRICTION = 0.85
}
```

### Как это работает:

Каждый кадр скорость умножается на 0.85 (остаётся 85% от предыдущей):
- Кадр 1: `velocity = 5` м/сек
- Кадр 2: `velocity = 5 * 0.85 = 4.25` м/сек
- Кадр 3: `velocity = 4.25 * 0.85 = 3.6` м/сек
- ...
- Кадр 20: `velocity ≈ 0.1` м/сек (почти остановился)

**Результат:** плавная остановка вместо моментальной

---

## 6. ОБНОВЛЕНИЕ ПОЗИЦИИ

```daslang
let current_pos = nodeId.localPosition
let new_pos = current_pos + velocity * get_delta_time()
nodeId.localPosition = new_pos
```

### Параметры:

- **`nodeId.localPosition`** — текущая позиция (float3: x, y, z)
- **`velocity`** — текущая скорость (float3: единиц/сек)
- **`get_delta_time()`** — время с последнего кадра в секундах
  - На 60 FPS это ≈ 0.0167 сек
  - На 30 FPS это ≈ 0.033 сек

### Формула движения:

```
новая_позиция = текущая_позиция + скорость × время_кадра

Пример на 60 FPS (delta = 0.0167 сек):
- velocity = (5, 0, 0) м/сек (движение вправо)
- delta_time = 0.0167 сек
- перемещение = (5, 0, 0) × 0.0167 = (0.0835, 0, 0) м
- За 1 секунду (60 кадров): 0.0835 × 60 = 5 метров ✓
```

## ПАРАМЕТРЫ НАСТРОЙКИ

Внутри класса `SPlayer`:

```daslang
MOVE_SPEED : float = 5.0   // Максимальная скорость (единиц/сек)
FRICTION : float = 0.85    // Коэффициент трения (0-1)
```

### Как изменить:

- **Увеличить скорость:** `MOVE_SPEED = 10.0`
- **Уменьшить скорость:** `MOVE_SPEED = 2.0`
- **Быстрее тормозить:** `FRICTION = 0.7` (остаётся 70%)
- **Медленнее тормозить:** `FRICTION = 0.95` (остаётся 95%)

## ОСНОВНЫЕ ФУНКЦИИ

| Функция | Назначение |
|---------|-----------|
| `add_and_activate_action_set()` | Создать и активировать систему управления |
| `get_action_vector2(name)` | Получить вектор ввода (-1..1, -1..1) |
| `get_delta_time()` | Получить время прошедшее с последнего кадра |
| `normalize(vector)` | Сделать вектор единичной длины |
| `length(vector)` | Получить длину вектора |
| `nodeId.localPosition` | Получить/установить локальную позицию ноды |

## ЛОГИКА ПОТОКА

```
┌─ on_initialize() ─┐
│ Настраиваем Action Set для WASD
│
├─ on_update() (каждый кадр) ─┐
│ 1. Получаем ввод: move_input = get_action_vector2("move")
│ 2. Преобразуем в 3D: move_direction = float3(move_input.x, 0, move_input.y)
│ 3. Применяем скорость или трение:
│    - Если ввод: normalize(move_direction) × MOVE_SPEED
│    - Если нет: velocity × FRICTION
│ 4. Обновляем позицию: nodeId.localPosition += velocity × delta_time
│
└─ Результат: Плавное WASD управление ─┘
```

## ТЕСТИРОВАНИЕ

1. Запустите игру (игра загружается)
2. Нажимайте **W, A, S, D** — куб должен двигаться по плоскости
3. Отпустите клавиши — куб должен плавно останавливаться
4. Нажимайте **W+D** (диагональ) — куб должен двигаться с той же скоростью как при одной клавише

## ПРИМЕЧАНИЯ

- Это базовое горизонтальное движение **без прыжка**
- Y-координата всегда = 0 (игрок не летает)
- Система работает через Action Set (не прямая проверка клавиш)
- Для добавления прыжка нужно будет добавить логику на Y-ось и использовать физику
